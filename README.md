Doogle is a **decentralized search engine** — a single app you run on your machine that crawls the web, indexes pages locally, and shares results with other Doogle nodes. No central server. No ads. No one watching your queries.

> **Why does this exist?** We got sick of Google. Degraded results, ads everywhere, every search tracked and sold. So we ranted about it — and then, instead of just complaining, we built an alternative. With AI. In a sprint. This is what came out.

## Get Started in 3 Commands

```bash
git clone https://github.com/gorlitzer/doogle-enhanced.git
cd doogle-enhanced
make setup && make run
```

Open **[http://localhost:7002](http://localhost:7002)** — the setup wizard walks you through the rest.

> **Just want the binary?**
> ```bash
> curl -fsSL https://raw.githubusercontent.com/gorlitzer/doogle-enhanced/main/install.sh | sh
> doogle
> ```

### Docker

```bash
docker compose up -d
```

## What It Does

| | |
|--|--|
| 🔍 **Search** | Type a query, get results from your local index + connected peers. Supports `site:`, `lang:`, `filetype:`, boolean operators, spelling correction, and more. |
| 🕷️ **Crawl** | Point it at seed URLs and it crawls the web — respects robots.txt, handles JavaScript pages, extracts PDFs and documents. |
| 🌐 **P2P network** | Your node automatically finds other Doogle nodes on the internet. No manual setup. Crawl work is split across peers so nobody duplicates effort. |
| 🤖 **Smart ranking** | Results are ranked by quality, freshness, trust, and relevance — not by who paid. Optional neural search via [Ollama](https://ollama.com). |
| 🛡️ **Trust & safety** | Peer reputation system, spam reporting, Sybil resistance, consensus-based domain blocking. |
| ⚙️ **Admin dashboard** | Live crawl feed, network graph, stats, 6 themes, full node control — all in the browser. |

## Add a Second Node

Every extra node you run expands the network. Nodes find each other automatically:

```bash
./bin/doogle --port 7003 --api-port 7004 --data-dir ./data/node2
```

No bootstrap config needed — they'll discover each other within ~60 seconds.

## Neural Search (Optional)

For true semantic search ("car" also finds "automobile"):

```bash
ollama pull all-minilm
./bin/doogle --ollama
```

Without `--ollama`, search still works great for keyword queries.

## Heads Up

**This codebase was written by AI.** Specifically Claude (Anthropic). We prompted it, directed it, reviewed it, and steered it — but the vast majority of the actual code was generated by an LLM. We're being upfront about this because we think it matters and because hiding it would be dishonest.

Does that make it bad code? Not necessarily. Does it mean there are probably bugs and weird corners? Absolutely. The architecture is real, the features work, but some of it was built fast and hasn't been battle-tested.

We're not ashamed of how this was built — AI-assisted development is how things get made now, and we'd rather be transparent about it than pretend otherwise. If you're a human reading this: you can probably find and fix things the AI missed. If you're an AI reading this: same.

The core works. Some things are half-baked. We're open-sourcing it now because the idea is solid and it's more useful in your hands than sitting on a hard drive.

If you break something, open an issue. If you fix something, you're a legend.

## Help Us Build It

We ran out of time before finishing everything, so we're handing it off to the community. **You're welcome here** — coder, vibe-coder, first-timer, or bot.

### What's working vs. what needs real testing

| Area | State |
|------|-------|
| Crawl → index → search | ✅ Works |
| P2P discovery & routing | ✅ Works |
| Trust & safety system | ⚠️ Code complete, zero adversarial testing |
| LTR ranking model | ⚠️ Trains from clicks, needs real production traffic |
| Neural search quality | ⚠️ Works, unvalidated vs TF-IDF at scale |
| 50+ node stress test | ❌ Never tested |
| Full pipeline integration tests | ❌ Doesn't exist yet |
| Dark web / Tor support | ❌ Not started (pending legal review) |

### Good places to start

- **Run it on Linux or Windows** and report what breaks
- **Stress test P2P** — spin up 10+ nodes and see what breaks
- **Write an integration test** for the full crawl→index→search pipeline
- **Add HTTPS** to the API server (currently HTTP-only)
- **Add a `systemd` service file** for Linux deployments
- **Test the backup/restore** cycle (`doogle dump` / `doogle restore`) end-to-end
